K8S项目实践(07): kubeadm安装k8s集群(containerd版)

By mospan

2024年01月15日

1. 规划

使用 kubeadm 安装 Kubernetes 集群并使用 containerd 作为容器运行时（container runtime）是一种常见的安装方法。

OS 配置 用途 备注
aliOS(172.17.197.69) 2核(vCPU) 4GiB 5 Mbps k8s-master1 乌兰察布 可用区 C
aliOS(172.17.197.68) 4核(vCPU) 10 GiB 5 Mbps GPU：NVIDIA T4/8 k8s-worker1 乌兰察布 可用区 C

注：这是演示 k8s 集群安装的实验环境，配置较低，生产环境中我们的服务器配置至少都是 8C/16G 的基础配置。

2. 版本选择

  • Alibaba Cloud Linux 3.2104 LTS 64位

2.1. 配置主机名

所有节点执行

[root@k8s-master1 ~]# hostnamectl set-hostname k8s-master1
[root@k8s-worker1 ~]# hostnamectl set-hostname k8s-worker1

所有节点执行

 # 关闭firewalld

2.5. 加载 br_netfilter 模块

确保 br_netfilter 模块被加载 > 所有节点执行

# 加载模块
root@k8s-master1 ~]# modprobe br_netfilter
## 查看加载请看
[root@k8s-master1 ~]# lsmod | grep br_netfilter
br_netfilter           32768  0
bridge                270336  1 br_netfilter

# 永久生效
[root@k8s-master1 ~]# cat <<EOF | tee /etc/modules-load.d/k8s.conf
> br_netfilter
> EOF
br_netfilter
[root@k8s-master1 ~]#

2.6. 允许 iptables 检查桥接流量

所有节点执行

[root@k8s-master1 ~]# cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
> net.bridge.bridge-nf-call-ip6tables = 1
> net.bridge.bridge-nf-call-iptables = 1
> EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
[root@k8s-master1 ~]# sysctl --system

2.7. 关闭 swap

所有节点执行

# 临时关闭
[root@k8s-master1 ~]# swapoff -a

# 永久关闭
[root@k8s-master1 ~]# sed -ri 's/.*swap.*/#&/' /etc/fstab

2.8. 安装ipset及ipvsadm

安装ipset及ipvsadm

[root@k8s-master1 ~]# dnf install -y ipset ipvsadm

配置ipvsadm模块加载方式，添加需要加载的模块

$ cat > /etc/sysconfig/modules/ipvs.modules <<EOF
#!/bin/bash
modprobe -- ip_vs
modprobe -- ip_vs_rr
modprobe -- ip_vs_wrr
modprobe -- ip_vs_sh
modprobe -- nf_conntrack
EOF

授权、运行、检查是否加载

$ chmod 755 /etc/sysconfig/modules/ipvs.modules && bash /etc/sysconfig/modules/ipvs.modules && lsmod | grep -e ip_vs -e nf_conntrack

3. 安装containerd

所有节点执行

3.1. 安装 containerd

通过 yum 安装 containerd：

root@k8s-master1 ~]# yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
Adding repo from: http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
[root@k8s-master1 ~]# yum install containerd -y

3.2. 配置 containerd

$ /usr/bin/containerd config default > /etc/containerd/config.toml
$ sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml
$ sed -i 's/sandbox_image = "registry.k8s.io\/pause:3.6"/sandbox_image = "registry.aliyuncs.com\/google_containers\/pause:3.9"/g' /etc/containerd/config.toml

注： 记得确保配置修改成功，尤其是pause版本。

3.3. 启动containerd

启动并使 containerd 随系统启动：

root@k8s-master1 ~]# sudo systemctl start containerd
root@k8s-master1 ~]# sudo systemctl enable containerd
root@k8s-master1 ~]# ps -ef | grep containerd
root       37760       1  0 07:35 ?        00:00:00 /usr/bin/containerd
root       37807   36202  0 07:35 pts/0    00:00:00 grep --color=auto containerd
[root@k8s-master1 ~]#

说明containerd已启动成功。

5. master节点

5.1. k8s 初始化

Master 节点执行

注：记得修改–apiserver-advertise-address与–kubernetes-version修改正确。

root@k8s-master ~]# kubeadm init \
  --apiserver-advertise-address=172.17.197.69 \
  --image-repository registry.aliyuncs.com/google_containers \
  --kubernetes-version v1.28.15 \
  --service-cidr=10.96.0.0/12 \
  --pod-network-cidr=10.244.0.0/16 \
  --ignore-preflight-errors=all \
  --cri-socket /var/run/containerd/containerd.sock

参数说明：

   --apiserver-advertise-address  # 集群master地址
   --image-repository             # 指定k8s镜像仓库地址
   --kubernetes-version           # 指定K8s版本（与kubeadm、kubelet版本保持一致）
   --service-cidr                 # Pod统一访问入口
   --pod-network-cidr             # Pod网络（与CNI网络保持一致）

初始化后输出如下内容，说明成功。

.....

To start using your cluster, you need to run the following as a regular user:

  mkdir -p $HOME/.kube

5.5. 容器网络（CNI）部署

Master 节点执行

插件地址：https://kubernetes.io/docs/concepts/cluster-administration/addons/

该地址在 k8s-master 初始化成功时打印出来。

5.5.1. 选择一个主流的容器网络插件部署（Calico）

5.5.2. 下载yml文件

wget https://docs.projectcalico.org/manifests/calico.yaml --no-check-certificate

5.5.3. 看看该yaml文件所需要启动的容器

[root@k8s-master1 install-k8s-cluster]# cat calico.yaml |grep image
          image: docker.io/calico/cni:v3.25.0
          imagePullPolicy: IfNotPresent
          image: docker.io/calico/cni:v3.25.0
          imagePullPolicy: IfNotPresent
          image: docker.io/calico/node:v3.25.0
          imagePullPolicy: IfNotPresent
          image: docker.io/calico/node:v3.25.0
          imagePullPolicy: IfNotPresent
          image: docker.io/calico/kube-controllers:v3.25.0
          imagePullPolicy: IfNotPresent
[root@k8s-master1 install-k8s-cluster]#

5.5.4. 修改为国内可下载镜像

使用docker.m.daocloud.io代替docker.io

sed -i s/docker.io/docker.m.daocloud.io/g calico.yaml 
$ cat calico.yaml | grep image

5.5.5. 安装calico

[root@k8s-master1 calicodir]# kubectl  apply -f calico.yaml

5.5.6. 查看pod和node状态正常

[root@k8s-master1 calicodir]# kubectl  get pod -A
[root@k8s-master1 calicodir]# kubectl  get node


可以看出pod已全部Running， node已变成Ready状态，说明master节点已安装成功完成。
